# Security Policy

If you wish to report a security vulnerability but not in a public issue -- thank you! -- I ask that you follow the following process.

Use the email security AT timnash.co.uk.

Describe the vulnerability.

If you have a fix, that is most welcome -- please attach or summarize it in your message!

I will evaluate the vulnerability and, if necessary, release a fix or mitigating steps to address it. I will contact you to let you know the outcome, and will credit you in the report.

Please do not disclose the vulnerability publicly until a fix is released!

Once I have either a) published a fix, or b) declined to address the vulnerability for whatever reason, you are free to publicly disclose it.

A plea from me (Tim), this is a personal project repo, it's a one person project developed with an open source license. I do not have a budget for a bug bounty, I appreciate vulnerability reports and act on them in good faith please don't submit a vulnerability report expecting financial compensation.
